# NPICK Core Logic

NPICK의 추천은 45개 숫자 중 6개를 균등 확률로 뽑는 단순 랜덤이 아니다. 역대 로또 6/45 1등 당첨번호에서 번호별 점수를 만들고, 그 점수로 가중 추첨한 뒤 검증 조건을 통과한 조합만 추천한다.

## 데이터 입력

- `data/lotto-data.js`의 역대 1등 당첨번호를 사용한다.
- 기본 분석 범위는 최근 260회이며, 사용자가 슬라이더로 범위를 바꿀 수 있다.
- 데이터 갱신은 `node scripts/update-lotto-data.mjs`로 수행한다.

## 통계 계산

`src/core/statistics.js`는 선택된 분석 범위에서 다음 값을 계산한다.

- 번호별 출현 횟수
- 번호별 마지막 출현 이후 간격
- 자주 나온 번호, 적게 나온 번호, 오래 안 나온 번호
- 과거 1등 당첨 조합 Set
- 당첨번호 합계의 하위 18%, 상위 82% 분위 범위

## 추천 생성

`src/core/recommendation-engine.js`는 전략별 가중치를 계산한다.

- 균형형: 극단적으로 많이 또는 적게 나온 번호보다 중간 빈도, 출현 빈도, 미출현 간격을 섞는다.
- 빈도형: 자주 나온 번호의 비중을 높이고 미출현 간격을 일부 반영한다.
- 역발상형: 적게 나온 번호와 오래 안 나온 번호의 비중을 높인다.
- 혼합형: 균형형, 빈도형, 역발상형을 추천 세트별로 순환한다.

최근 5회에 나온 번호는 사용자가 옵션을 켠 경우 가중치를 낮춘다. 번호 선택은 가중 추첨이므로 무작위성은 남아 있지만 모든 번호가 같은 확률로 선택되지는 않는다.

## 검증 조건

후보 조합은 아래 조건을 통과해야 한다.

- 6개 번호가 모두 서로 달라야 한다.
- 옵션이 켜진 경우 과거 1등 당첨 조합과 같으면 제외한다.
- 번호 합계가 분석 범위의 18%~82% 분위 밖이면 제외한다.
- 연속번호가 4개 이상이면 제외한다.
- 균형 옵션이 켜진 경우 홀짝 비율은 2:4~4:2 범위여야 한다.
- 균형 옵션이 켜진 경우 저고 비율은 1~22와 23~45 기준 2:4~4:2 범위여야 한다.
- 균형 옵션이 켜진 경우 같은 10번대 구간에 4개 이상 몰리면 제외한다.

## 설계 원칙

- 추천 알고리즘은 DOM을 모른다.
- 통계 계산은 렌더링을 모른다.
- UI 렌더링은 추천 점수 계산을 모른다.
- 앱 초기화 파일은 상태 연결과 이벤트 바인딩만 담당한다.
- 핵심 추천 로직을 변경할 때는 이 문서를 함께 갱신한다.
